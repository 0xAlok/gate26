5. How to Study Effectively

This methodology helped someone go from rejections to landing an internship in 9 months. Don't just consume content - study in depth, not blackbox:

  1. 1
    Gather ALL resources first (15-20 min) - Before starting any topic, collect: the original paper, blog posts, YouTube videos, GitHub code, documentation.
  2. 2
    Study each resource in detail - Don't skim. Actually read the paper, watch the full video, run the code.
  3. 3
    Understand the WHY - Don't blackbox anything. Ask "Why does this work? What happens if I change X? What problem does this solve?"
  4. 4
    Write to retain - Blog about what you learned. Explain it to others. Teaching is the best way to learn.
  5. 5
    Revise periodically - Come back to topics. Spaced repetition beats cramming.

"I changed my way of studying and studied a lot in depth stuff rather than blackboxing stuff" - This mindset shift turned rejections into offers.

6. Coding Best Practices (Don't Skip These!)

From engineers who learned these the hard way. These habits separate hobbyists from professionals:

Look at Inputs & Outputs

Always know: What does the function take? What does it return? Is it np.ndarray or torch.Tensor? Is it in-place? This is good engineering, not just ML.

Type Hints & Docstrings

Use Python type hints (PEP 484+) even if you're the only one reading the code. Write docstrings that actually explain what the function does.

Make It Work, Then Refactor

Don't waste time on clean code if it doesn't work. Get it working first, then worry about abstraction. Premature optimization is the root of all evil.

Write Tests Religiously

TDD makes you think about inputs, return types, edge-cases. You'll catch bugs before they crash production. Start with pytest.

Ask More Questions

Ask on Reddit, StackOverflow, LinkedIn, X. Ask questions nobody is asking. More you ask, more you learn. Don't worry about looking dumb.

Read the Documentation

"Don't buy a $500 course. Read the API documentation. It's free and it's better than any guru's video." - Build boring stuff first.
